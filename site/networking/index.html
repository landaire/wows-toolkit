How Connections Work

Sessions are end-to-end encrypted using iroh to manage connections. When connecting to other people in a session, they join using a session token like: 

toolkit-eNoNycEBwCAIA8CVkIDWcQyY_Uew9z3F_ECMzbKFrmqPHIorIZUmqihX4HLmoh_YP-s4OrjvA1SEEm4

This token only contains an EndpointId of the session host: 

$ echo 'eNoNycEBwCAIA8CVkIDWcQyY_Uew9z3F_ECMzbKFrmqPHIorIZUmqihX4HLmoh_YP-s4OrjvA1SEEm4' \
  | base64 -d | crabz -d -f zlib

f4683b319bc073dccd2451f4eff35f50fbfcbf2f43eb657b2a30f357a23d4b9e

Connection Establishment

The EndpointId is a cryptographic identifier which is used to connect to the host. To establish this connection, a relay server is used to find your peer and relay communications to them until a direct peer-to-peer connection can be established.

Once a direct connection is negotiated, all traffic flows directly between peers over QUIC—the relay is no longer in the path. If a direct connection cannot be established (e.g. due to restrictive NAT), the relay continues to forward encrypted traffic. In either case, the relay server cannot read the contents of your messages because the connection is end-to-end encrypted.

tl;dr: Connections are secure and use a central relay to establish the initial connection until a direct connection between peers in the session can be established.

Protocol Messages

All messages are serialized with rkyv (zero-copy deserialization) and framed with a 4-byte length prefix. Permission enforcement is client-side: each receiver decides what to accept based on the sender's role.

Handshake

Peer → Host, then Host → Peer
Join / SessionInfo Peer sends version + display name; host responds with the peer list, permissions, and current session state.

Session Sync

Any peer ↔ All peers
Cursors & Pings Live minimap cursor positions and map pings shared between all peers.
Annotations Drawing annotations (lines, circles, freehand strokes, etc.) synced in real time.
Heartbeat Sent every 10s per connection. 30s without any message = connection considered dead.

Replay Streaming

Frame source → All peers
Frames Zlib-compressed draw commands streamed from whoever has the replay open.
Playback & Display Play/pause, speed, and render options (ship names, range rings, trails, etc.).

Host Authority

Host / Co-host → All peers
Permissions Lock or unlock annotations and display settings for regular peers.
Roster UserJoined / UserLeft notifications and co-host promotion.
